1. 프로그램 소개

이 프로그램은 인기 모바일 게임 '프린세스 커넥트! Re:Dive'(이하 프리코네)의 아레나(배틀 아레나, 프린세스 아레나)에서 상대방의 조합을 보고 효과적인 공략 조합을 빠르게 찾을 수 있도록 도와주는 자동화 도구입니다.

게임 화면에서 상대방 조합 캐릭터를 마우스로 드래그하여 캡처하면, 프로그램이 자동으로 캐릭터 아이콘을 인식하고, 이를 바탕으로 pcrdfans.com 웹사이트에서 최적의 공격 조합을 검색해줍니다.

주요 기능:

  • 화면 드래그 캡처: 게임 내 조합을 손쉽게 캡처합니다.
  • 캐릭터 자동 인식: OpenCV 기술을 활용하여 캡처된 이미지에서 캐릭터 아이콘을 자동으로 감지하고 분리합니다.
  • 다단계 유사도 비교: 캡처된 아이콘과 로컬/웹사이트 아이콘 간의 다단계 비교를 통해 정확도를 높입니다. (★6 아이콘 자동 ★3 변환 포함)
  • 웹사이트 자동 검색: Selenium을 사용하여 pcrdfans.com에서 인식된 조합으로 자동 검색을 실행합니다.
  • 다국어 및 서버 지원: 웹 사이트 영어, 일본어, 중국어 설정 및 다양한 서버별 결과 필터링을 지원합니다.

2. 시작하기 전 준비사항

  • Python 설치: Python 스크립트로 프로그램이 제공되었다면 반드시 설치해야 합니다. (8. 자주 묻는 질문의 Q6 참고 ) (단, 실행 파일(.exe) 형태로 제공되었다면 설치 불필요)

  • Google Chrome 브라우저: 이 프로그램은 Chrome 브라우저를 사용하여 웹 자동화를 수행합니다. 반드시 Chrome이 설치되어 있어야 하며, 최신 버전으로 유지하는 것을 권장합니다.
  • 안정적인 인터넷 연결: pcrdfans.com 및 캐릭터 아이콘 데이터 다운로드를 위해 필요합니다. Wi-Fi 환경 등 안정적인 네트워크에서 사용하시는 것이 좋습니다.

영상 4-6: 아레나 조합 자동 검색 실행 과정 (1배속)

  • 프로그램은 캡처된 아이콘을 순서대로 분석합니다. 로그 창에 각 아이콘 처리 과정이 간략히 표시됩니다.
    1. 1단계 (인게임 아이콘 비교): 캡처된 아이콘과 로컬에 저장된 전체 인게임 캐릭터 아이콘(JPG)을 비교하여 가장 유사한 캐릭터를 1차 식별합니다.
    2. 2단계 (★6 → ★3 자동 변환): 만약 1단계에서 식별된 캐릭터가 ★6이면, 웹사이트 검색에 주로 사용되는 ★3 아이콘으로 자동 변환합니다. (예: 쿄우카(★6) 아이콘 → 쿄우카(★3) 아이콘)
    3. 3단계 (웹사이트 아이콘 비교): 2단계를 거친 (또는 원래의) 인게임 아이콘을 pcrdfans.com에서 사용하는 스프라이트 아이콘과 다시 비교하여 최종적으로 웹사이트에서 클릭할 아이콘을 확정합니다.
  • 모든 아이콘이 식별되면, 프로그램이 자동으로 Chrome 브라우저를 제어하여 pcrdfans.com 웹사이트에 접속합니다 (이미 열려있다면 해당 탭 활용).
  • 선택된 언어, 서버, 정렬 방식 설정에 맞춰 웹사이트 필터를 조정한 후, 식별된 캐릭터 아이콘을 순서대로 클릭합니다.
  • 마지막으로 웹사이트의 검색 버튼을 자동으로 클릭하여 공략 조합 결과를 보여줍니다.

검색이 완료되면 로그 창에 관련 메시지가 표시됩니다. 브라우저 창은 사용자가 직접 닫아야 합니다.

  2. 필요시 임계값을 조절합니다. (테스트 결과에 임계값이 반영됩니다)
  3. [유사도 비교 테스트] 버튼을 클릭합니다.
    • 메인 창의 로그 영역에 각 분할 이미지별 상세 분석 과정(1단계 인게임 아이콘 매칭 결과, 2단계 ★6 → ★3 변환 여부, 3단계 스프라이트 아이콘 매칭 결과 상위 후보들)이 텍스트로 출력됩니다.
    • 분석이 완료되면, 새로운 창 ("유사도 비교 테스트 결과")이 나타나 시각적인 비교 결과를 보여줍니다.
      • 상단: 원본 캡처 이미지 미리보기 (사용자가 드래그한 전체 영역)
      • 하단 표:
        • 분할 이미지: 프로그램이 자동 감지/분리한 각 아이콘
        • 인게임 아이콘: 1단계에서 가장 유사하다고 판단된 로컬 JPG 아이콘과 그 유사도 점수
        • ★6 → ★3 전환: 1단계 아이콘이 ★6일 경우, 변환된 ★3 아이콘 (해당 없거나 실패 시 다르게 표시)
        • 유사도 1~3위: 3단계에서 pcrdfans 스프라이트 아이콘과 비교하여 가장 유사하다고 판단된 상위 3개 아이콘과 각각의 유사도 점수, 그리고 어떤 로컬 JPG 아이콘을 기준으로 비교했는지 표시됩니다.

7. 주요 용어 설명

  • OpenCV (Open Source Computer Vision Library): 이미지 처리 및 컴퓨터 비전 기능을 제공하는 오픈소스 라이브러리입니다. 이 프로그램에서는 캡처된 이미지에서 캐릭터 아이콘의 윤곽선을 감지하는 데 사용됩니다.
  • Selenium: 웹 브라우저를 자동화하는 데 사용되는 도구입니다. 이 프로그램에서는 pcrdfans.com 웹사이트에 자동으로 접속하고, 캐릭터 아이콘을 클릭하며, 검색을 실행하는 데 사용됩니다.
  • WebDriver (ChromeDriver): Selenium이 특정 웹 브라우저(이 경우 Chrome)를 제어할 수 있도록 하는 인터페이스 또는 프로그램입니다. webdriver-manager는 이 ChromeDriver를 자동으로 관리(다운로드, 버전 관리)해줍니다.
  • SSIM (Structural Similarity Index Measure): 두 이미지 간의 구조적 유사도를 측정하는 지표입니다. 값은 보통 -1에서 1 사이이며, 1에 가까울수록 두 이미지가 시각적으로 더 유사함을 의미합니다. 이 프로그램에서는 캡처된 아이콘과 저장된 아이콘 간의 유사도를 비교하여 동일 캐릭터인지 판단하는 기준으로 사용됩니다. ("임계값"은 이 SSIM 점수를 기준으로 합니다.)
  • 스프라이트 시트 (Sprite Sheet): 여러 개의 작은 이미지를 하나의 큰 이미지 파일에 모아 놓은 것입니다. pcrdfans.com은 캐릭터 아이콘을 이 방식으로 관리하며, 프로그램은 이 스프라이트 시트를 다운로드하여 개별 아이콘으로 분리합니다.
  • 인게임 아이콘 (In-game Icon): 실제 프리코네 게임 내에서 사용되는 캐릭터 아이콘을 의미합니다. 이 프로그램에서는 redive.estertion.win에서 다운로드한 JPG 파일들이 이에 해당합니다.
  • ROI (Region of Interest, 관심 영역): 이미지 전체가 아닌, 분석에 중요한 특정 부분 영역을 의미합니다. 이 프로그램에서는 캐릭터 아이콘 이미지에서 실제 캐릭터 얼굴 및 주요 특징이 포함된 핵심 영역만을 선택적으로 비교하여 유사도 계산의 정확성을 높입니다.
  • 임계값 (Threshold): 어떤 판단을 내리기 위한 기준값을 의미합니다. 이 프로그램에서는 SSIM 점수가 사용자가 설정한 임계값 이상이어야 두 아이콘을 동일 캐릭터로 간주합니다.
  • GUI (Graphical User Interface): 사용자가 프로그램을 시각적인 요소(버튼, 창, 메뉴 등)를 통해 쉽게 조작할 수 있도록 하는 인터페이스입니다.

8. 자주 묻는 질문

Q1: 이 프로그램은 무료인가요?

A1: 네, 이 프로그램은 무료로 제공됩니다.


Q2: 어떤 운영체제에서 사용할 수 있나요?

A2: 이 프로그램은 Python으로 작성되었으며, Google Chrome 브라우저가 설치된 Windows 환경에서 테스트 및 개발되었습니다. 이론적으로는 Python 및 필요한 라이브러리가 설치된 다른 OS(macOS, Linux)에서도 실행 가능할 수 있으나, 화면 캡처, 창 제어 등 일부 기능은 OS에 따라 다르게 동작하거나 호환성 문제가 발생할 수 있습니다. 공식적으로는 Windows 환경을 권장합니다.


Q3: 프로그램 사용에 특별한 기술이 필요한가요?

A3: 기본적인 컴퓨터 사용 능력과 프리코네 게임에 대한 이해만 있으면 충분합니다. 이 설명서를 따라 하시면 쉽게 사용하실 수 있습니다.


Q4: 이 프로그램이 제 게임 계정에 영향을 주거나, 제재를 받을 가능성은 없나요?

A4: 이 프로그램은 게임 화면을 캡처하고, 공개된 웹사이트(pcrdfans.com)의 정보를 활용하여 검색을 자동화하는 방식으로 동작합니다. 게임 클라이언트를 직접 수정하거나, 서버와 비정상적인 통신을 하지 않으므로 일반적인 사용으로 인한 계정 제재 위험은 매우 낮다고 판단됩니다. 또한, 이 프로그램은 사용자의 게임 계정 정보나 개인 정보를 요구하거나 수집하지 않습니다. 하지만 모든 자동화 도구 사용에는 항상 잠재적인 위험이 따를 수 있음을 인지하시고, 사용자 본인의 판단과 책임하에 사용하시기 바랍니다. (본 설명서 상단의 "중요 숙지사항" 및 14. 면책 조항 참고)


Q5: 프리코네 게임이 아닌 다른 게임에서도 사용할 수 있나요?

A5: 아니요, 이 프로그램은 프리코네 아레나의 캐릭터 아이콘과 pcrdfans.com 웹사이트 구조에 특화되어 개발되었으므로 다른 게임에서는 사용할 수 없습니다.


Q6: 프로그램을 어떻게 설치하나요?

A6: 개발자의 배포 방식에 따라 다릅니다.

  • Python 스크립트 형태로 제공:
    1. Python 최신 버전 (예: 3.8 이상)을 설치합니다. (python.org)
      (※ Python 설치 과정에서 "Add Python to PATH" 옵션을 반드시 활성화 하시기 바랍니다.)

    2. 프로그램 실행에 필요한 라이브러리들을 설치합니다. 프로그램 폴더에 requirements.txt 파일이 있다면, 터미널(명령 프롬프트 또는 PowerShell)에서 해당 폴더로 이동 후 pip install -r requirements.txt 명령어를 실행합니다. 
      pip install -r requirements.txt

      만약 requirements.txt 파일이 없다면, 코드 상단에 명시된 라이브러리(예: cv2, Pillow, scikit-image, requests, selenium, webdriver-manager, beautifulsoup4, numpy)들을 pip install 라이브러리명 형태로 개별 설치합니다.
      (주의: tkinter는 Python 표준 라이브러리이므로 pip로 별도 설치할 필요가 없습니다.)

      pip install opencv-python Pillow scikit-image selenium webdriver-manager requests beautifulsoup4 numpy

      ❗ 패키지 설치 중 오류 발생 시: pip를 최신 버전으로 업그레이드한 후 다시 시도해 보세요. 
      python -m pip install --upgrade pip


    3. 메인 Python 스크립트 파일(예: arena_searcher.py)을 실행합니다.

  • 실행 파일(.exe) 형태로 제공: 별도의 설치 과정 없이 다운로드한 파일을 실행하면 됩니다.

Q7: "webdriver-manager", "selenium" 등의 오류가 발생하며 실행되지 않아요.

A7: Q6에서 안내된 대로 프로그램 실행에 필요한 라이브러리가 제대로 설치되지 않았거나, Python 또는 라이브러리 버전 간 호환성 문제일 수 있습니다. 설치된 라이브러리 버전을 확인하고, 필요시 재설치하거나 버전을 조정해 보세요. pip list 명령어로 설치된 라이브러리 목록 및 버전을 확인할 수 있습니다. 특히, selenium과 webdriver-manager는 최신 버전으로 유지하는 것이 좋습니다. 로그 창의 오류 메시지를 확인하여 어떤 라이브러리에서 문제가 발생하는지 파악하는 것이 중요합니다.


Q8: Chrome 브라우저가 꼭 필요한가요? 다른 브라우저는 안 되나요?

A8: 네, 현재 프로그램은 Google Chrome 브라우저 자동화를 기준으로 작성되었습니다 (ChromeDriverManager 사용). 다른 브라우저(Firefox, Edge 등)를 지원하려면 코드 수정이 필요하며, 현재 버전에서는 지원하지 않습니다. Chrome이 설치되어 있지 않거나, 프로그램이 Chrome 실행 파일을 찾을 수 없는 경우 (예: 특이한 경로에 설치) 오류가 발생합니다.


Q9: "사이트 연결" 버튼은 언제 눌러야 하나요? 매번 눌러야 하나요?

A9: 한 번 성공적으로 연결하고 아이콘 데이터를 받아오면, 프로그램 재시작 전까지는 일반적으로 다시 누르지 않아도 됩니다. 하지만 브라우저 창을 사용자가 임의로 닫았거나, 인터넷 연결 문제로 웹사이트와의 연결이 끊어졌다고 판단될 경우 다시 눌러야 합니다. 로그 창에 관련 오류가 표시될 수 있습니다.


Q10: 아이콘 다운로드는 얼마나 자주 해야 하나요?

A10: "[사이트 연결]" 시 자동으로 최신 버전을 확인하고 필요시 업데이트합니다. 따라서 사용자가 직접 신경 쓸 필요는 거의 없습니다.


Q11: "전체 캐릭터 아이콘 다운로드"가 너무 오래 걸려요. 정상인가요?

A11: 수백 개의 캐릭터 아이콘을 다운로드하고 WebP에서 JPG로 변환하는 과정이므로 인터넷 속도 및 PC 성능에 따라 몇 분(일반적으로 1분 이내, 경우에 따라 그 이상) 정도 소요될 수 있습니다. 이는 정상적인 과정이며, 한 번 완료하면 다시 할 필요는 거의 없습니다. 로그 창에 개별 파일 다운로드 및 변환 진행 상황이 표시되니 잠시 기다려주세요. 만약 비정상적으로 오래 걸리거나(예: 3분 이상) 멈춘 것 같다면 인터넷 연결을 확인하고 프로그램을 재시작하여 다시 시도해 보세요.


Q12: 아이콘 파일은 어디에 저장되나요? 임의로 삭제해도 되나요?

A12: 아이콘 파일은 프로그램 실행 폴더 내의 character_images 폴더에 하위 폴더(예: 20250503와 같은 버전명 폴더, character_unit_icon_all 폴더)를 만들어 저장됩니다. 이 파일들은 프로그램 작동에 필수적이므로 임의로 삭제하거나 수정하면 프로그램이 정상 작동하지 않을 수 있습니다. 만약 문제가 발생하여 삭제해야 한다면, 삭제 후 프로그램을 재시작하여 "[사이트 연결]" 또는 "[ (수동) 전체 캐릭터 아이콘 다운로드 ]"를 통해 다시 받아야 합니다.


Q13: pcrdfans.com의 아이콘과 redive.estertion.win의 아이콘을 둘 다 사용하는 이유가 무엇인가요?

A13: 이 프로그램은 정확하고 효율적인 아이콘 인식 및 웹 자동화를 위해 다단계 비교 방식을 사용합니다.

  • 1단계: redive.estertion.win 아이콘 (인게임 아이콘, JPG): 사용자가 캡처한 게임 화면의 아이콘과 가장 먼저 비교됩니다. 실제 인게임 아이콘과 매우 유사하여 어떤 캐릭터인지 1차적으로 식별하는 데 가장 정확도가 높습니다. 이 단계는 "이 아이콘은 대략 어떤 캐릭터일 것이다"라는 후보군을 찾는 과정입니다.
  • 2단계: ★6 → ★3 변환: 1단계에서 ★6 캐릭터로 식별된 경우, 대부분의 웹사이트 검색(pcrdfans 포함)에서는 기본 등급(주로 ★3) 아이콘을 사용하므로, 해당 캐릭터의 ★3 아이콘으로 내부적으로 변환하여 다음 단계 비교의 정확도를 높입니다.
  • 3단계: pcrdfans.com 아이콘 (스프라이트 아이콘, PNG): 1, 2단계를 거쳐 최종 식별/변환된 인게임 아이콘을 pcrdfans.com 웹사이트에서 실제로 클릭해야 할 대상 아이콘과 다시 한번 비교합니다. 이를 통해 웹사이트의 수많은 아이콘 중 정확히 클릭해야 할 아이콘을 최종 확정합니다. 이 단계는 "후보군 중에서 웹사이트의 이 아이콘이 확실하다"고 결정하는 과정입니다.

Q14: "아레나 조합 캡쳐" 시 어떤 부분을 캡처해야 하나요?

A14: 상대방 조합의 캐릭터 얼굴 또는 아이콘 전체가 명확하게 보이도록, 아이콘 영역을 모두 포함하여 드래그해야 합니다. 아이콘 테두리까지 충분히 포함하되, 각 아이콘 사이의 간격은 최소화하는 것이 좋습니다. 너무 넓거나 좁게 캡처하면 인식률이 떨어질 수 있습니다. (4.1 아레나 조합 캡처의 스크린샷 예시 참고)


Q15: 캡처 영역을 잘못 지정했어요. 취소할 수 있나요?

A15: 현재 버전에서는 드래그 중 선택을 취소하는 기능은 없습니다. 마우스 버튼을 놓으면 해당 영역이 캡처되므로, 만약 잘못 선택했다면 당황하지 마시고 프로그램 메인 창으로 돌아온 후 다시 [아레나 조합 캡쳐] 버튼을 눌러 재시도하시면 됩니다.


Q16: 듀얼 모니터(또는 다중 모니터)를 사용하는데, 캡처가 제대로 안 돼요.

A16: 프로그램은 일반적으로 주 모니터를 기준으로 화면 캡처를 시도합니다. 프리코네 게임이 실행 중인 모니터가 Windows 설정에서 주 모니터로 설정되어 있는지 확인해 보세요. 만약 문제가 지속된다면, 일시적으로 단일 모니터 환경으로 변경하여 사용하거나, 프리코네 게임 창을 주 모니터로 옮겨 실행하는 것을 권장합니다. 일부 시스템에서는 창 모드로 실행 중인 게임의 캡처가 더 안정적일 수 있습니다.


Q17: 캡처했는데 아이콘 미리보기에 아무것도 안 떠요 / 이상한 이미지가 떠요.

A17: 이 문제는 다양한 원인이 있을 수 있습니다. 다음 사항들을 점검해 보세요.

  • 캡처 영역 확인: 4.1 아레나 조합 캡처의 가이드라인대로 정확히 캡처했는지 다시 확인해 주세요. 캐릭터 아이콘이 아닌 배경이나 다른 UI를 캡처했을 수 있습니다.
  • 화면 가림 확인: 게임 화면 위에 다른 창(메신저 알림, 다른 프로그램 등)이 순간적으로 나타나 가리고 있었을 수 있습니다. 캡처 시에는 게임 화면 외에 다른 창이 없도록 하는 것이 좋습니다.
  • OpenCV 인식 실패: 매우 드물지만, 특정 화면 환경이나 게임 내 특수 효과(예: 캐릭터 등장 연출) 등으로 인해 OpenCV가 아이콘 윤곽선을 제대로 감지하지 못했을 수 있습니다. 프로그램 재시작이나 PC 재부팅이 도움이 될 수 있습니다.
  • 유사도 비교 테스트 활용: 5.1 유사도 비교 테스트를 실행하여 원본 캡처 이미지가 어떻게 보이는지, 프로그램이 어떤 부분을 아이콘으로 인식하려 했는지(또는 못했는지) 확인해 보세요.

Q18: 캐릭터 인식이 잘 안 돼요. 다른 캐릭터로 인식하거나 아예 인식을 못 해요.

A18: 이 문제는 다양한 원인이 있을 수 있습니다. 다음 사항들을 점검해 보세요.

  • 캡처 품질: 선명하고 정확하게 캡처되었는지 확인하세요. (Q14, Q17 참고)
  • 임계값 조절: 4.2 캡처된 아이콘 확인 및 임계값 설정 및 9. 고급 활용팁을 참고하여 임계값을 조절해 보세요.
  • 아이콘 데이터 최신 상태: "[사이트 연결]"을 통해 pcrdfans.com 아이콘을 최신화하고, 필요시 "[ (수동) 전체 캐릭터 아이콘 다운로드 ]"를 통해 redive.estertion.win 아이콘도 최신 데이터로 유지하세요. 특히 게임 업데이트로 신규 캐릭터나 신규 ★6이 추가된 경우 데이터 업데이트가 필수적입니다.
  • 유사도 비교 테스트 활용: 5.1 유사도 비교 테스트를 사용하여 어떤 아이콘과 혼동되는지, 각 단계별 유사도 점수는 어느 정도인지 상세히 확인하여 원인을 분석할 수 있습니다.
  • ★6 아이콘 문제: ★6 아이콘은 테두리나 배경 효과가 ★3과 달라 인식에 영향을 줄 수 있습니다. 프로그램은 이를 자동으로 ★3 기준으로 변환하여 비교하려고 시도하지만, 완벽하지 않을 수 있습니다. 특히 신규 ★6 캐릭터의 경우 데이터 업데이트가 필요할 수 있습니다.
  • 특정 캐릭터의 어려움: 일부 캐릭터는 아이콘 디자인 자체가 다른 캐릭터와 매우 유사하거나(예: 동일 캐릭터의 다른 버전), 배경이 복잡하거나, 아이콘 크기가 유독 작거나 커서 인식률이 낮을 수 있습니다.

Q19: 한정 캐릭터나 콜라보 캐릭터도 인식하나요?

A19: redive.estertion.win과 pcrdfans.com에서 해당 캐릭터의 아이콘 데이터를 제공하고, 사용자의 프로그램에 해당 아이콘 데이터가 최신 버전으로 다운로드되어 있다면 인식 가능합니다. 최신 캐릭터의 경우, 아이콘 데이터가 각 사이트에 반영되고 사용자가 프로그램 내에서 데이터를 업데이트할 때까지 시간이 걸릴 수 있습니다.


Q20: "아레나 조합 검색" 버튼을 눌러도 아무 반응이 없거나, Chrome 브라우저가 안 떠요.

A20: 이 문제는 다양한 원인이 있을 수 있습니다. 다음 사항들을 점검해 보세요.

  • 사이트 연결 확인: 3.3 사이트 연결 및 아이콘 초기화가 성공적으로 완료되었는지 확인하세요. Selenium WebDriver(Chrome)가 제대로 초기화되지 않았을 수 있습니다. 로그 창에 "사이트 연결 및 아이콘 처리 완료!" 메시지가 있는지 확인하세요.
  • 로그 창 확인: 로그 창에 특정 오류 메시지(예: WebDriverException, NoSuchElementException 등)가 있는지 확인하세요. 해당 오류 메시지를 검색하거나 개발자에게 문의하면 해결에 도움이 될 수 있습니다.
  • Chrome 설치 및 버전 확인: Google Chrome이 올바르게 설치되어 있고, 최신 버전인지 확인하세요. Chrome 브라우저 자체가 손상되었을 수도 있습니다. 필요시 Chrome을 재설치해 보세요.
  • 백신/방화벽 확인: 일부 백신 프로그램이나 방화벽이 Chrome 또는 WebDriver의 실행을 차단하고 있지 않은지 확인하세요. 프로그램 또는 관련 프로세스(예: chromedriver.exe)를 예외 처리해야 할 수 있습니다.
  • 캡처된 아이콘 부재: [아레나 조합 캡쳐]를 통해 아이콘을 먼저 캡처해야 합니다. 메인 창의 미리보기 영역에 아이콘이 모두 (또는 인식 가능한 만큼) 표시되어 있어야 검색이 진행됩니다.

Q21: "유사도 비교 테스트" 결과 창의 점수는 무엇을 의미하나요?

A21: 점수는 SSIM(Structural Similarity Index Measure, 구조적 유사도 지수) 값을 의미하며, 일반적으로 -1에서 1 사이의 값을 가집니다. 이 프로그램에서는 0과 1 사이로 정규화되어 표시될 수 있으며, 1에 가까울수록 두 이미지가 시각적으로 더 유사하다는 뜻입니다. 프로그램은 사용자가 설정한 "임계값"보다 이 SSIM 점수가 높아야 두 아이콘을 동일(또는 매우 유사한) 캐릭터로 판단합니다. 예를 들어, 임계값이 0.50일 때 어떤 아이콘 비교 점수가 0.65라면 유사하다고 판단하고, 0.45라면 다르다고 판단합니다.


Q22: 프로그램 업데이트는 어떻게 하나요?

A22: 개발자가 새로운 버전의 스크립트나 실행 파일을 배포하면, 기존 파일을 새 파일로 교체하여 업데이트할 수 있습니다.

  • 업데이트 시 character_images 폴더는 가급적 유지하시는 것이 좋습니다. 이전 버전에서 다운로드한 아이콘 데이터를 새 버전에서도 대부분 사용할 수 있어 다시 다운로드하는 시간을 절약할 수 있습니다.
  • 단, 프로그램의 아이콘 처리 방식에 큰 변경이 있거나, 개발자가 배포 시 별도로 character_images 폴더 삭제를 권장하는 경우에는 해당 폴더를 백업 후 삭제하고 새 버전에서 아이콘을 다시 다운로드하는 것이 좋을 수 있습니다. (보통은 기존 폴더 유지)

Q23: 개발자에게 버그를 제보하거나 기능을 제안하고 싶어요.

A23: 매우 환영합니다! 11. 피드백 및 버그 제보 안내 섹션을 참고하여 자세한 정보를 제공해 주시면 프로그램 개선에 큰 도움이 됩니다.


9. 고급 활용팁

  • 프린세스 아레나(프레나) 공략 시:
    • 이 프로그램은 한 번에 한 조합만 분석합니다. 프린세스 아레나의 경우, 상대방의 1, 2, 3번 각 조합마다 개별적으로 [아레나 조합 캡쳐] 및 [아레나 조합 검색] 과정을 반복해야 합니다. 각 조합 검색 후 브라우저의 탭을 유지하거나 결과를 메모해두면 편리합니다.

  • 임계값 조절 심층 가이드:
    • 기본값(0.50) 활용: 대부분의 경우 기본 임계값으로 충분합니다. 먼저 기본값으로 시도해 보세요.
    • 특정 캐릭터 오인식 시: 만약 특정 캐릭터(예: 배경이 매우 복잡한 아이콘, 변신 전/후 모습이 아이콘 상 크게 다른 캐릭터, 색감이 유사한 다른 캐릭터)가 자주 오인식된다면, 5.1 유사도 비교 테스트 기능을 활용하세요.
      1. 오인식되는 상황을 재현하여 캡처하고, [유사도 비교 테스트]를 실행합니다.
      2. 결과 창에서 해당 분할 아이콘에 대해 실제로 일치하는 캐릭터의 스프라이트 아이콘 유사도 점수와 오인식된 캐릭터의 스프라이트 아이콘 유사도 점수를 비교합니다.
      3. 일치하는 캐릭터의 점수보다는 높고, 오인식된 캐릭터의 점수보다는 낮은 적절한 값으로 임계값을 미세 조정해 볼 수 있습니다. (예: 일치하는 캐릭터와의 유사도가 0.45, 오인식된 캐릭터와의 유사도가 0.55라면, 임계값을 0.48~0.52 사이로 조절)
    • 너무 낮은 임계값 주의: 임계값을 너무 낮추면(예: 0.30 이하) 전혀 다른 캐릭터도 유사하다고 판단하여 검색 결과의 질이 떨어질 수 있습니다. 오히려 더 많은 오인식이 발생할 수 있습니다.
    • 너무 높은 임계값 주의: 임계값을 너무 높이면(예: 0.70 이상) 캡처 시 약간의 각도나 조명 차이에도 다른 캐릭터로 인식하거나 아예 인식하지 못할 수 있습니다. 특히 다양한 환경에서 캡처해야 하는 경우 불리할 수 있습니다.

  • 빠른 재검색을 위한 준비:
    • 아레나 상위권에서는 비슷한 조합을 자주 만날 수 있습니다. 프로그램이 이미 실행 중이고 "[사이트 연결]"이 완료된 상태라면, 새로운 상대를 만났을 때 [아레나 조합 캡쳐] -> [아레나 조합 검색] 과정만 빠르게 반복하여 효율을 높일 수 있습니다. (현재 버전에서는 이전 캡처 결과 저장/로드 기능은 지원하지 않습니다.)

  • 네트워크 환경 최적화:
    • 아이콘 다운로드 및 웹사이트 자동 검색은 인터넷 연결 속도에 영향을 받습니다. 가능하면 유선 LAN 또는 안정적인 Wi-Fi 환경에서 사용하시는 것이 쾌적합니다. 모바일 핫스팟 등 불안정한 네트워크에서는 지연이나 오류가 발생할 수 있습니다.

  • 화면 해상도와 게임 창 크기:
    • 프로그램은 다양한 화면 해상도에서 작동하도록 설계되었으나, 너무 낮거나 비표준 해상도에서는 캡처나 아이콘 인식에 예기치 않은 문제가 발생할 수 있습니다. 게임 창이 너무 작으면 아이콘 해상도가 낮아져 인식률이 떨어질 수 있습니다.

10. 문제 해결 및 팁

  • "사이트 연결이 끊어져 있습니다." 또는 WebDriverException 관련 오류 발생 시:
    • 가장 먼저 인터넷 연결 상태를 확인하세요.
    • [사이트 연결] 버튼을 다시 클릭하여 연결을 재시도하세요.
    • Chrome 브라우저가 예기치 않게 종료되었거나 응답하지 않는 상태일 수 있습니다. 작업 관리자(Ctrl+Shift+Esc)에서 실행 중인 모든 chrome.exe 및 chromedriver.exe 프로세스를 종료하고 다시 시도해 보세요.
    • 드물게 webdriver-manager가 최신 ChromeDriver를 제대로 다운로드하지 못했거나, 기존 ChromeDriver 파일이 손상되었을 수 있습니다. C:\Users\사용자명\.wdm\drivers\chromedriver (Windows 기준, 사용자명은 실제 사용자 폴더명) 경로의 하위 폴더(버전명 폴더)를 삭제하고 프로그램을 재실행하면 ChromeDriver를 다시 다운로드합니다.

  • 캐릭터 아이콘 인식이 지속적으로 잘 안 될 경우 (Q18 반복 점검 후에도):
    • 게임 내 그래픽 설정 확인: 프리코네 게임 내 그래픽 설정이 너무 낮거나, 화면 필터(블루라이트 필터, 야간 모드 등 OS 또는 서드파티 프로그램 필터)가 적용되어 아이콘 색감이 크게 왜곡된 경우 인식률에 영향을 줄 수 있습니다. 게임 내 그래픽 설정을 '표준' 또는 '고화질'로, 외부 필터는 해제하고 시도해 보세요.
    • PC 성능 문제: PC 사양이 매우 낮아 게임 자체가 끊기거나 화면 반응이 느린 경우, 캡처 시 이미지가 흔들리거나 제대로 된 프레임을 잡지 못할 수 있습니다. 다른 리소스를 많이 사용하는 프로그램을 종료하고 시도해 보세요.

  • pcrdfans.com 웹사이트 구조 변경 시 대처:
    • 만약 pcrdfans.com 웹사이트의 HTML 구조(클래스 명, ID, XPATH 등)가 크게 변경되면 프로그램의 웹 자동화 로직(아이콘 클릭, 필터 적용, 검색 버튼 클릭 등)이 정상 작동하지 않을 수 있습니다. 이 경우 로그 창에 NoSuchElementException (요소를 찾을 수 없음), TimeoutException (요소가 나타날 때까지 기다렸지만 실패) 등의 오류가 자주 발생할 수 있습니다.
    • 이러한 문제는 사용자가 직접 해결하기 어려우며, 개발자의 프로그램 업데이트를 통해서만 해결될 수 있습니다. 11. 피드백 및 버그 제보 안내를 통해 개발자에게 알려주시면 프로그램 개선에 큰 도움이 됩니다.

  • "전체 캐릭터 아이콘 다운로드" 중 특정 파일 다운로드/변환 실패 시:
    • 로그 창에 어떤 파일에서 문제가 발생했는지 확인하세요. (예: 다운로드 실패: 123456.webp - HTTP 404, JPG 변환 실패: 123456.webp - (오류 내용))
    • redive.estertion.win 사이트의 일시적인 문제이거나 해당 파일 자체의 문제일 수 있습니다.
    • 몇몇 파일 실패는 전체 기능에 큰 영향을 주지 않을 수 있지만(해당 캐릭터를 사용하지 않는다면), 반복되거나 중요한 캐릭터에서 발생한다면 개발자에게 문의하는 것이 좋습니다.

  • 브라우저가 자동으로 닫히지 않는 것이 정상입니다:
    • 아레나 조합 검색이 완료된 후, 프로그램이 제어하던 Chrome 브라우저 창은 사용자가 검색 결과를 확인하고 활용할 수 있도록 자동으로 닫히지 않습니다. 사용자가 직접 닫아주셔야 합니다.

  • 프로그램이 갑자기 멈추거나 응답 없음 상태가 되는 경우:
    • 대량의 아이콘 다운로드나 복잡한 이미지 처리, 웹 자동화 작업 중 일시적으로 PC 리소스를 많이 사용할 때 발생할 수 있습니다. 대부분의 경우 잠시 기다리면(몇 초 ~ 몇 분) 작업이 완료되고 정상으로 돌아옵니다. 특히 "[사이트 연결]" 버튼 클릭 직후나 "[ (수동) 전체 캐릭터 아이콘 다운로드 ]" 실행 시 발생할 수 있습니다.
    • 만약 장시간 응답이 없다면, 작업 관리자를 통해 프로그램을 강제 종료하고 재시작해야 할 수 있습니다. 재시작 후에도 동일한 지점에서 문제가 반복된다면 로그를 확인하고 개발자에게 문의하세요.

  • 항상 프로그램과 Chrome 브라우저를 최신 버전으로 유지:
    • webdriver-manager는 대부분 최신 Chrome 버전에 맞는 ChromeDriver를 자동으로 관리해주지만, 간혹 Chrome 브라우저의 급격한 업데이트와 webdriver-manager의 ChromeDriver 지원 사이에 시간 차이가 발생하여 호환성 문제가 생길 수 있습니다. 프로그램과 Chrome 브라우저 모두 가능한 최신 안정화 버전으로 유지하는 것이 좋습니다.
